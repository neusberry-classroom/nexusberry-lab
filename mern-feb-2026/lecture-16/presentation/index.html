Part 1

Understanding the DOM — Selection & Traversal

The DOM is the browser's representation of your HTML as a tree of JavaScript objects.

💎 Blueprint vs Building: HTML is the blueprint. The DOM is the building. JavaScript rearranges the furniture.

The DOM Tree

document
html
head
body
header
main
footer

Every element is a node with parent, children, and sibling relationships

Selecting Elements

MethodReturnsSelector
getElementById("id")One elementID only
querySelector(".css")First matchAny CSS
querySelectorAll(".css")NodeList (all)Any CSS
⚡ Best Practice: Use querySelector / querySelectorAll for everything — CSS selector syntax you already know!

DOM Traversal


// Children
board.children;              // HTMLCollection of child elements
board.firstElementChild;     // First child element
board.lastElementChild;      // Last child element

// Parent & Siblings
card.parentElement;                 // Direct parent
column.nextElementSibling;         // Next sibling
column.previousElementSibling;     // Previous sibling

// closest() — find nearest ANCESTOR (walks UP)
const column = card.closest(".column");   // Finds parent column
const board = card.closest("#board");     // Any levels up
⚠ Always use closest() instead of chaining parentElement calls — resilient to structural changes.

What does each line return?


const a = document.querySelector(".card");

console.log(typeof a, typeof b);
  1. Both return an array of elements
  2. a = single Element (or null), b = NodeList ✓

Part 1 Summary

❌ Common Mistake: querySelector returns null when nothing matches — always check before accessing properties. Use el?.textContent
⚠ Watch Out: NodeList supports forEach but NOT map, filter, reduce. Spread to Array: [...nodeList]
💎 Pro Insight: Cache DOM queries in variables — don't call querySelector repeatedly for the same element.

Part 2

Creating, Modifying & Removing Elements

JavaScript constantly adds, modifies, and removes DOM elements — this is how interactive UIs work.

Creating Elements — Two Approaches

innerHTML

Quick & readable

⚠ XSS risk with user input
⚠ Destroys event listeners

VS

createElement

Safe & efficient

✓ Preserves listeners
✓ Returns reference

⚡ Rule: innerHTML for content you control. createElement for user data.

classList — Surgical Class Management


element.classList.add("active");         // Add
element.classList.remove("active");      // Remove
element.classList.toggle("active");      // Toggle on/off
element.classList.contains("active");    // Check → boolean
element.classList.replace("old", "new"); // Swap
❌ Never: element.className = "active" — this overwrites ALL classes! Use classList methods.

data-* Attributes & dataset

element.dataset.id;          // "42"
element.dataset.priority;    // "high"
element.dataset.createdAt;   // "2026-04-01" (kebab → camelCase)

⚡ Pro Tip: data-* attributes let elements carry their own metadata — the DOM equivalent of encapsulation.

Let's see this in code!

Building a createTaskCard helper function


function createTaskCard(id, title, desc, priority = "medium") {
  const card = document.createElement("div");
  card.classList.add("card", `priority-${priority}`);
  card.setAttribute("draggable", "true");
  card.dataset.id = id;
  card.dataset.priority = priority;
  card.innerHTML = `
    <strong>${title}</strong>
    <p>${desc}</p>
    <button data-action="delete">✕</button>
  `;
  return card;
}
todoColumn.append(createTaskCard(4, "Deploy", "Push to Vercel", "high"));

Part 2 — Interactive Question

What's the security risk?


const userName = document.getElementById("nameInput").value;
document.getElementById("greeting").innerHTML = `Welcome, ${userName}!`;
  1. No risk — innerHTML is always safe
  2. Performance issue — innerHTML is slow
  3. XSS attack! User can inject <script> tags. Use textContent instead ✓
  4. TypeError — innerHTML doesn't accept template literals

Part 2 Summary

❌ Common Mistake: innerHTML += destroys all existing event listeners. Use append() or insertAdjacentHTML() instead.
⚠ Watch Out: Inline styles use camelCase: element.style.fontSize, not font-size.
💎 Pro Insight: React's JSX compiles to createElement calls. Understanding this tells you what React does under the hood.

Part 3

Events — Listeners, Delegation & The Event Object

Events are how your code reacts to user interactions — the Observer pattern applied to the DOM.

Event Flow — Capture, Target, Bubble

📄 document
↓
💻 body
↓
📁 .column
↓
🎯 .card (TARGET)

① Capture (down) → ② Target → ③ Bubble (back up) — listeners fire during bubbling by default

Event Delegation

❌ Without Delegation


cards.forEach(card => {
  card.addEventListener(
    "click", handleClick
  );
});
// New cards have NO listener!
VS

✓ With Delegation


board.addEventListener("click",
  (e) => {
    const card = e.target
      .closest(".card");
    if (card) handleClick(card);
});
// Works for ALL cards!

The Event Object

PropertyPurpose
event.targetElement that was clicked (source)
event.currentTargetElement the listener is on
event.type"click", "submit", "keydown", etc.
event.preventDefault()Stop default behavior
event.stopPropagation()Stop bubbling
⚠ target ≠ currentTarget when using delegation!

Let's see this in code!

Event delegation on the task board


board.addEventListener("click", (event) => {
  // Check if a delete button was clicked
  const deleteBtn = event.target.closest('[data-action="delete"]');
  if (deleteBtn) {
    const card = deleteBtn.closest(".card");
    card.remove();
    return;
  }

  // Check if a card was clicked
  const card = event.target.closest(".card");
  if (card) {
    card.classList.toggle("selected");
    return;
  }
});

Common Event Types

EventFires WhenElement
clickMouse click or touchAny
inputValue changes (every keystroke)Input
changeValue changes (on blur)Input, select
submitForm submittedForm
keydown / keyupKey pressed / releasedAny
focus / blurGains / loses focusInput

Part 3 — Interactive Question

Listener on board, user clicks <strong> inside a card. What are target and currentTarget?

  1. target: DIV (board), currentTarget: STRONG
  2. Both are STRONG
  3. target: STRONG (clicked), currentTarget: DIV (board, listener) ✓

Part 3 Summary

❌ Common Mistake: Anonymous arrow functions can't be removed with removeEventListener — store handlers in named variables.
⚡ Best Practice: Always use event delegation for lists, grids, and dynamic content. Use closest() inside handlers.
💎 Pro Insight: React uses event delegation at its core — one root listener for the entire app. You just learned React's event architecture.

Part 4

Task Board Capstone — Drag & Drop, Forms & Dynamic UI

Putting it all together: forms, drag & drop, data attributes, and DOM performance.

HTML5 Drag & Drop Events

✋ dragstart Pick up
→
🙋 dragover Over zone
→
📦 drop Release
→
✅ dragend Cleanup
⚠ Key: You MUST call event.preventDefault() on dragover to allow dropping! The browser rejects drops by default.

DocumentFragment — Batch DOM Updates

❌ Direct Append

100 items = 100 reflows


for (item of items) {
  const el = createElement("div");
  container.append(el);
  // Reflow every time!
}
VS

✓ DocumentFragment

100 items = 1 reflow


const frag = document
  .createDocumentFragment();
for (item of items) {
  const el = createElement("div");
  frag.append(el);
}
container.append(frag); // 1 reflow!

Let's see this in code!

Form submission + input validation


form.addEventListener("submit", (event) => {
  event.preventDefault();  // Stop page reload!

  const title = document.getElementById("taskInput").value.trim();
  if (!title) return;  // Guard clause

  // Validate with regex (from L13)
  if (!/^[a-zA-Z0-9\s\-_.,!?]+$/.test(title)) {
    console.warn("Invalid characters");
    return;
  }

  const newCard = createTaskCard(nextId++, title, "New task", priority);
  todoColumn.append(newCard);
  input.value = "";
  input.focus();
});

Part 4 — Interactive Question

We remove preventDefault() from the dragover handler. What happens?

  1. Drop works but the card moves to the wrong position
  2. Drop silently fails — the drop event never fires ✓
  3. A JavaScript error is thrown
  4. The card is deleted from the DOM

Part 4 Summary

❌ Common Mistake: Forgetting preventDefault() on dragover — drops silently fail. Most counterintuitive DOM behavior.
⚡ Best Practice: Use DocumentFragment when inserting 3+ elements. Batch styles with classList instead of individual style.* changes.
💎 Pro Insight: React batches all DOM updates automatically — today you did manually what React does under the hood.
